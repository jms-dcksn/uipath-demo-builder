# Shared References

These references define consistency rules reused by all partitions.

## Files

- `mapping-conventions.md`: naming, statuses, IDs, and traceability rules.
- `demo-design-principles.md`: practical guidance for high-impact demo UX design.
- `DemoScriptReference.docx`: example narrated script showing message-to-visual storytelling.
- `demo-script-authoring-notes.md`: distilled script structure and authoring checklist.
- `research-and-citation-rules.md`: source quality expectations and citation format.

## Usage

- Read these before filling any templates.
- Keep one canonical convention set; avoid re-defining rules in partition docs.
- If you change a convention, document the rationale and date.
