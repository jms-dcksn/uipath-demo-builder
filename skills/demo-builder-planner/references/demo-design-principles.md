# Demo Design Principles

## 1) Start With Business Value

- Frame each screen around a decision or outcome.
- Prioritize what proves value quickly in a live demo.

## 2) Show Realistic Workflows

- Include both happy path and exception path.
- Demonstrate clear human/automation collaboration points.
- Make execution type visible (`AI Agent`, `RPA`, `IDP`, `API`, `Human Task`) where it supports the narrative.

## 3) Keep Data Credible

- Use realistic but anonymized example records.
- Keep field relationships coherent across list and detail pages.

## 4) Make Interactions Obvious

- Dashboard should answer: "What needs attention now?"
- Detail page should answer: "What decision should I make now?"

## 5) Optimize For Storytelling

- Pre-stage a small set of varied cases (normal, urgent, exception).
- Make state transitions visible after user actions.

## 6) Build For Reusability

- Reuse shared conventions and template sections.
- Capture assumptions and tradeoffs as you design.

## 7) Script The Demo Explicitly

- Define `3-4` key messages tied to business outcomes.
- Map each key message to `2-3` visuals that prove the point.
- For every beat, write narrator line + operator action + visible outcome.
- Progress the story from triage -> investigation/orchestration -> decision/confidence.
- End with measurable impact language, not feature recap.
